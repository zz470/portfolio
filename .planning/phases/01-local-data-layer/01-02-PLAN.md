3. Reports any mismatches (data file references a path that does not exist, or an SVG exists that is not referenced)
4. Exits with code 0 if all paths match, code 1 if any mismatch

Run the script and ensure it passes. Delete the script after running — it should not remain in the repo.

Example bash approach:
```bash
# Extract thumbnail_url and hero_url values from the data file
grep -oP '(?:thumbnail_url|hero_url): "([^"]+)"' lib/data/projects.ts | \
  sed 's/.*: "//' | sed 's/"//' | while read -r url; do
    file="public${url}"
    if [ ! -f "$file" ]; then
      echo "MISSING: $file"
      exit 1
    fi
  done
```
